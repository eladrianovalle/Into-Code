Setting up your Ghost blog for free on GitHub Pages

Saturday, March 07, 2015

I did this following these resources:
this this this , but had some trouble and thought that a more detailed, step-by-step guide would benefit others who are at a similar skill level as I.

Much of the work here will be done on the command line. Follow these steps carefully and you shouldn't have any problems.

Install Node.js

Use Homebrew

Download and Install Ghost

Install locally

Check latest version at Ghost.org

Make Ghost directory

mkdir ghost

Move into the directory

cd ghost

Start Ghost

npm start

Login to Ghost

This command will open in your browser. This will connect you to the Ghost Admin interface. Create an account now.

open http://localhost:2368/ghost

After you've created your account, you'll see your dashboard. Here you'll find an pre-generated blog post that details how to write your posts using Markdown. Take time to take a look.

Create a GitHub account

Create account. Free.

Create your GitHub Pages repo

See the GitHub Pages Basics Documentation for details.

You must use the username/username.github.io naming scheme. The repo name must be lower case even if your username has upper case letters.

Install Buster

With the tool Buster you can export the Ghost blogs into static pages. First we install Buster with:

brew install python  
sudo pip install buster

Deploy

Make sure that Ghost is running locally.

Prepare static folder

Buster requires a folder specifically named static in which it will generate your static pages from your locally running instance of Ghost.

I have placed the static folder for Buster inside my Ghost installation. So I just cloned my username/username.github.io repo with a target directory name like this:

git clone git@github.com:username/username.github.io.git static

This will link your GitHub repo to this folder. Later we will have Buster generate your blog and push it onto this repo.

Write Blog posts

Now your setup is complete and you can start writing Blog posts in Ghost. Make sure Ghost is still running. If not, simply move into the ghost directory and start running it.

cd ghost
npm start

Now you can work on your blog. Configure it to your heart's desire and write some posts.

Export your Ghost blog

All published blog posts could be exported to the static folder with Buster.

buster generate --domain=http://localhost:2368

This command will take the product of your locally running instance of Ghost, your blog, and generate the files needed to push the static pages up to GitHub. Remember, Ghost needs to be running for this step to work. If you get any errors like Connecting to localhost|::1|:2368... failed: Connection refused., this is most probably the reason. Since no instance of Ghost is currently running at localhost:2368, Buster has nothing to connect to, and the connection fails.

buster preview  
open http://localhost:9000

The output of Buster can be previewed on port 9000. As long as Ghost is running, you'll be able to keep refreshing the preview in your browser to immediately see your last changes. This allows you to view your blog as it will appear once it has been deployed to the web prior to it being made live. This way you can check for errors, work on your formatting, etc, and make sure you're satisfied before you go live.

Deploy to GitHub Pages

buster deploy

This will add, commit and push all files in the static folder to your GitHub repo.

For the initial push, please wait up to 10 minutes until GitHub deploys your subdomain.

All upcoming pushes are much faster and you can see your static Ghost blog posts on https://username.github.io

Write new Blog posts

To write new Blog posts or update the existing ones, first start Ghost. 

cd ghost  
npm start

Now you can go to http://localhost:2368/ghost to access your dashboard. Here you can create new posts, edit posts, or tweak the settings and appearance of your blog. Once you're done, use deploy to generate and deploy the static pages.

buster deploy

The sed commands will fix the share links from localhost to your correct GitHub pages URL.

That's it.
